This guide walks you through setting up the project, configuring the database connection, and running a full import of the INE `.DAT` fixed-width files.

## Prerequisites

Before you begin, make sure you have the following installed and available:

- **MySQL Server** — a running instance accessible over the network (the default config targets `192.168.56.101:3306`)
- **Java JDK 8 or later**
- **IntelliJ IDEA** (or any IDE that supports Maven/Gradle Java projects)
- **MySQL Connector/J** — the official JDBC driver for MySQL ([download from MySQL](https://dev.mysql.com/downloads/connector/j/))
- The INE election ZIP file: `02201606_MESA.zip`

<Steps>
  <Step title="Clone the repo and open the project">
    Clone the repository to your local machine and open it in your IDE.

    ```bash
    git clone <repository-url>
    ```

    In IntelliJ IDEA, go to **File → Open** and select the project root directory. Make sure the MySQL Connector/J `.jar` is added to the project's classpath (under **File → Project Structure → Libraries**).
  </Step>

  <Step title="Set up the MySQL database">
    Connect to your MySQL server and run the schema script to create all required tables.

    ```bash
    mysql -u perepi -p mydb < "Model SQL BD/Taules_eleccions.sql"
    ```

    This creates the following tables:

    | Table | Key columns |
    |---|---|
    | `comunitats_autonomes` | `comunitat_aut_id`, `nom`, `codi_ine` |
    | `provincies` | `provincia_id`, `comunitat_aut_id`, `nom`, `codi_ine`, `num_escons` |
    | `municipis` | `municipi_id`, `nom`, `codi_ine`, `provincia_id`, `districte` |
    | `eleccions` | `eleccio_id`, `nom`, `data`, `any`, `mes` |
    | `persones` | `persona_id`, `nom`, `cog1`, `cog2`, `sexe`, `data_naixement`, `dni` |
    | `candidatures` | `candidatura_id`, `eleccio_id`, `codi_candidatura`, `nom_curt`, `nom_llarg` |
    | `candidats` | `candidat_id`, `candidatura_id`, `persona_id`, `provincia_id`, `num_ordre`, `tipus` |
    | `eleccions_municipis` | `eleccio_id`, `municipi_id`, `num_meses`, `cens`, `vots_emesos`, … |
    | `vots_candidatures_mun` | `eleccio_id`, `municipi_id`, `candidatura_id`, `vots` |
    | `vots_candidatures_prov` | `provincia_id`, `candidatura_id`, `vots`, `candidats_obtinguts` |
    | `vots_candidatures_ca` | `comunitat_autonoma_id`, `candidatura_id`, `vots` |

    <Note>
      Ensure the database `mydb` exists before running the script. Create it with `CREATE DATABASE mydb;` if needed.
    </Note>
  </Step>

  <Step title="Configure the JDBC connection in Main.java">
    Open `src/basedades/Main.java` and update the connection string, username, and password to match your MySQL environment.

    ```java
    con = DriverManager.getConnection("jdbc:mysql://192.168.56.101:3306/mydb", "perepi", "pastanaga");
    ```

    Replace `192.168.56.101` with the IP address or hostname of your MySQL server. The default port is `3306`. The default database is `mydb`.

    <Warning>
      Do not commit credentials to version control. Consider externalising them to environment variables or a configuration file.
    </Warning>
  </Step>

  <Step title="Place the ZIP file in the ./zips/ directory">
    The importer expects the source ZIP at a fixed path. Create the `zips` directory at the project root if it does not already exist, then copy your file there.

    ```bash
    mkdir -p zips
    cp /path/to/02201606_MESA.zip zips/
    ```

    The extraction code looks for this exact filename:

    ```java
    String fileName = "./zips/02201606_MESA.zip";
    ```

    <Note>
      The importer also requires an empty `fitxers/` directory at the project root. Create it with `mkdir -p fitxers` if it does not exist — extracted `.DAT` files are written there before import.
    </Note>
  </Step>

  <Step title="Run Main.main()">
    Execute the `main` method in `basedades.Main`. In IntelliJ IDEA, right-click `Main.java` and select **Run 'Main.main()'**.

    The program runs in two phases:

    1. **Extraction** — `TractarFitxers.descomprimirDATsZip()` reads `02201606_MESA.zip`, iterates every entry, and copies any file ending in `.DAT` into `./fitxers/`.
    2. **Import** — `Importacions` methods run sequentially, each parsing the corresponding `.DAT` file and inserting rows into MySQL.

    The import order is fixed and matches foreign-key dependencies:

    ```java
    Importacions.importarComunitatsAutonomes();
    Importacions.importarProvincies();
    Importacions.importarMunicipis();
    Importacions.importarEleccions();
    Importacions.importarEleccionsMunicipi();
    Importacions.importarPersones();
    Importacions.importarCandidatures();
    Importacions.importarCandidats();
    Importacions.importarVotsProvincies();
    Importacions.importarVotsCandidatures_ca();
    Importacions.importarVotsMunicipis();
    ```

    Progress messages are printed to stdout as each `.DAT` file is extracted and each table is populated.
  </Step>

  <Step title="Verify the import">
    Once the program exits without errors, connect to MySQL and run a quick sanity check.

    ```sql
    -- Count imported autonomous communities
    SELECT COUNT(*) FROM comunitats_autonomes;

    -- Check total valid votes per province
    SELECT p.nom, SUM(vcp.vots) AS total_vots
    FROM vots_candidatures_prov vcp
    JOIN provincies p ON p.provincia_id = vcp.provincia_id
    GROUP BY p.nom
    ORDER BY total_vots DESC;
    ```

    A successful import populates all 11 tables with data from the 2016 Spanish general election.

    <Tip>
      If a table is empty, check that the corresponding `.DAT` file was extracted correctly into `./fitxers/` and that no exception was printed to the console during the import phase.
    </Tip>
  </Step>
</Steps>
