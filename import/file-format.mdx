---
title: File format
description: Reference for the INE fixed-width .DAT file format used as input to the Eleccions Generals import pipeline.
---

All source data comes from the Spanish National Statistics Institute (INE) as fixed-width plain-text files with a `.DAT` extension. Each line in a file represents one record. Fields are not delimited — they are read by character position and length.

## The `llegirSegonsLlargada` utility

All field extraction is handled by a single private helper method in `Importacions.java`:

```java
private static String llegirSegonsLlargada(int longiInci, int llargadaALlegir, String st) {
    String stringTornar = "";
    longiInci--;
    for (int i = 0; i < llargadaALlegir; ++i, ++longiInci) {
        stringTornar += String.valueOf(st.charAt(longiInci));
    }
    return stringTornar;
}
```

| Parameter | Description |
|-----------|-------------|
| `longiInci` | Start position of the field, **1-indexed**. The method subtracts 1 internally before accessing the character array. |
| `llargadaALlegir` | Number of characters to read. |
| `st` | The full text of the current line being processed. |

The return value is always a `String`. Callers cast to `int` with `Integer.parseInt()` where a numeric value is needed, and call `.trim()` to strip trailing spaces from text fields.

## Special parsing rules

**Catalan name connector.** When the first surname (`cog1`) read from `04021606.DAT` equals `"i"`, the connector is actually the conjunction between a compound first name and the real surname. In this case, the code splits `nom` into the actual first name and treats the second word as `cog1`.

**District filtering.** Both `importarMunicipis()` and `importarEleccionsMunicipi()` process `05021606.DAT` but only insert rows where `districte == 99`. These rows represent the whole municipality, not individual sub-districts. `importarVotsMunicipis()` and `importarVotsProvincies()` apply the same filter with their respective district fields.

**Null birth dates.** In `importarPersones()`, if the year field reads `0` the `data_naixement` column is set to `NULL` instead of constructing an invalid date.

**Community vs. province rows in 07021606.DAT.** A row is imported as an autonomous community when `codi_ine == 99` and `codi_ine_comunitat != 99`. A row is imported as a province when both codes are not `99`.

**Community-level votes in 08021606.DAT.** `importarVotsCandidatures_ca()` imports only rows where `comunitat_autonoma_ine != 99` AND `provincia_ine == 99`, i.e. the aggregated community totals rather than individual province breakdowns.
