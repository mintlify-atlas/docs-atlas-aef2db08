## Database tables referenced

All queries run against the `mydb` MySQL database. The tables most frequently referenced across the query library are:

| Table | Contents |
|---|---|
| `persones` | Personal data for candidates: name, surnames, DNI, date of birth, gender |
| `candidatures` | Political parties and coalitions, including short and long names |
| `candidats` | Assignments linking a person to a party and province for a given election |
| `provincies` | The 52 Spanish provinces with INE codes and seat (`num_escons`) allocations |
| `municipis` | Municipalities, each linked to a province |
| `eleccions_municipis` | Per-municipality vote totals: valid votes, blank votes, null votes, polling stations |
| `vots_candidatures_prov` | Votes and seats won per party per province |
| `vots_candidatures_mun` | Votes per party per municipality |
| `vots_candidatures_ca` | Votes per party per autonomous community |

For the full schema including column definitions and foreign key relationships, see the [data model reference](/data-model/overview).

## MySQL version requirements

<Note>
This query library requires **MySQL 8.0 or later**. Categoria 4 uses window functions (`OVER`, `PARTITION BY`) which are not available in MySQL 5.x. The data model also uses `GENERATED` columns, which require MySQL 5.7+.
</Note>
